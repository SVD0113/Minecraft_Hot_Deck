# Minecraft_Hot_Dack
스트림 덱 기능을 하는 마인크래프트 핫바를 PCB설계와 3D모델링을 통해 제품으로 만들고, AI를 활용해 소프트웨어 앱까지 프로그래밍.

Phase 1: 단일 하드웨어 실증(PoC) 및 물리 구조 설계
(진행 시기: 부품 배송 대기 중 / 현재)

SW 테스트 코딩: 현재 보유 중인 ESP32와 1.3인치 LCD 1개, 미니 텍트 스위치 1개를 빵판에 연결. 버튼을 누를 때마다 마인크래프트 다이아몬드 검 ➡️ 곡괭이 ➡️ 흙 블록 등으로 이미지가 즉각적으로 변환되는 C++(아두이노) 렌더링 코드를 작성하여 1차 구동 확인.

HW 물리 설계 구상 (핵심): 'LCD 화면 자체가 하나의 거대한 버튼'이 되는 구조(플로팅 스위치)를 설계.

실무 노하우: 버니어 캘리퍼스로 LCD 두께와 버튼 높이를 0.1mm 단위로 실측. "버튼을 PCB에 납땜했을 때, 그 위를 덮는 LCD가 위아래로 몇 mm를 움직여야 텐션감 있게 눌릴까?(공차/유격 계산)"를 가장 먼저 확정해야 함.

GitHub 기록 ([01_Idea_and_PoC] 폴더): 프로젝트 초기 블렌더 렌더링 샷, 기획안 텍스트, 알리익스프레스 부품 구매 이력(BOM 리스트), 빵판 단일 테스트 성공 영상 업로드.

Phase 2: 다중 제어 통신망 구축 및 PCB 기초 설계
(진행 시기: 배송 부품 도착 시점)

HW 다중 제어 검증: 74HC138 디코더와 추가 LCD 도착 즉시 빵판에 세팅. 디코더를 활용해 CS 핀이 없는 LCD 9개(기존 6개 + 신규 3개 혼합)를 각각 개별 제어하여, 9개 화면에 각기 다른 이미지를 띄우는 하드웨어 SPI 통신망 증명.

PCB 설계 기초 및 스케치: KiCad 기본 툴 사용법(회로도 그리기, 풋프린트 매칭) 학습. GitHub나 오픈소스 커뮤니티의 타사 ESP32/LCD 프로젝트 도면을 참고하여 내 제품에 맞는 회로도(Schematic) 구상.

Phase 3: PCB 설계 검증 및 외주 발주 (병목 구간)
(진행 시기: Phase 2 성공 직후)

PCB 레이아웃 배치: 회로도를 바탕으로 초록색 기판(PCB) 위에 부품 배치. Phase 1에서 계산한 간격을 적용해 9개의 LCD가 일렬로 예쁘고 정확하게 정렬되도록 최적화.

셀프 QA 및 검증 (가장 중요):

실무 노하우 1 (논리 검사): KiCad 내부의 설계 규칙 검사(DRC/ERC)를 돌려 선 끊김이나 쇼트 에러가 0이 될 때까지 간소화 및 수정.

실무 노하우 2 (물리 검사): 설계를 100% 비율로 A4 용지에 종이 프린트! 그 종이 위에 진짜 부품들을 올려보고, 나사 구멍 위치나 부품 간 겹침 현상이 없는지 외주 넣기 전 최종 더블 체크.

외주 발주: 검증이 끝난 거버(Gerber) 파일을 JLCPCB 등 공장에 제작 의뢰. 설계된 PCB는 3D 모델링용 뼈대로 쓰기 위해 .STEP 파일로 내보내기(Export) 저장.

GitHub 기록 ([02_Hardware_Design] 폴더): 완성된 회로도 PDF, PCB 레이아웃 렌더링 이미지, 핀 맵핑(Pinout) 요약 문서 커밋.

Phase 4: 기계 기구(3D) 설계 및 전용 소프트웨어(PC 앱) 개발
(진행 시기: PCB 기판 제작/배송 대기 기간)

3D 하우징 설계 및 공차 테스트:

Phase 3의 'PCB 3D 파일'을 라이노(Rhino)로 불러와 하판 뼈대로 고정하고, 그 위에 핫바 외관 껍데기 디자인.

실무 노하우: 처음부터 9칸 전체를 프린트하면 시간 낭비가 큼. 딱 '1칸짜리 모듈'만 테스트용으로 잘라서 3D 출력. LCD와 버튼이 부드럽게 눌리는지 확인 후 전체 하우징 출력.

전용 PC 소프트웨어(앱) 프로그래밍:

마인크래프트 메모리를 읽어 실시간 연동(체력/아이템), OBS 스트림덱 제어 등을 담당할 백그라운드 PC 클라이언트 앱(C# 또는 Python) 개발.

사용자 친화적 UI: 유저들이 원하는 이미지를 쉽게 넣고 빼며 커스텀 셋업을 서로 공유(Import/Export)할 수 있는 기능 추가. 타 게임으로의 확장성(API) 고려.

GitHub 기록 ([03_PC_Software] 폴더): PC 앱 소스 코드 업로드. 추후 유저 오픈소스를 위해 코드 모듈화 및 주석 철저히 작성.

Phase 5: 시스템 통합(SI) 조립 및 하드웨어 최종 테스트
(진행 시기: PCB 기판 도착 시점)

하드웨어 조립 및 납땜:

실무 노하우 (부분 테스트 납땜): 기판 도착했다고 9개 화면 다 납땜하면 안 됨! 전원부(캐패시터), ESP32 소켓, 디코더 칩 1개, LCD 1개만 먼저 납땜하고 전기를 넣어 쇼트가 없는지 확인. 정상 작동 시 나머지 부품 전체 납땜 진행.

시스템 통합 테스트:

완성된 기판을 3D 출력한 하우징에 조립. ESP32에 PC 연동 펌웨어를 올리고, 핫바 하드웨어 ↔ PC 앱 소프트웨어 간의 양방향 통신 총괄 테스트 진행.

Phase 6: 실사용 피드백, 레플리카 배포 및 포트폴리오 완성
(진행 시기: 완성본 도출 이후)

실사용 및 유지보수: 직접 책상에 두고 사용하며 버그, 버튼 키감, 화면 갱신 딜레이 등 수정/보완점 상세 기록.

베타 테스트 (지인 배포): 제품을 몇 개 더 복제(레플리카)하여 주변 스트리머/게이머 지인에게 배포.





노하우: 피드백은 구두로 듣지 말고 구글 폼(Google Forms)을 만들어 평점, 불편한 점 등을 정량적 데이터로 수집. 이를 바탕으로 v2.0 개선안 도출.

GitHub 마스터피스 완성 (README.md): 저장소 메인 화면에 핫바 작동 영상(GIF), 부품 구매처 링크, 3D 프린팅 껍데기 파일(.STL), PC 앱 다운로드 및 설치 가이드라인을 깔끔하게 문서화하여 완벽한 커리어용 포트폴리오로 퍼블리싱.
